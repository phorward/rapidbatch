The code generator
 
RapidBATCH Visual Dialog Designer allows the creation of the RapidBATCH script-code that is required to generate the dialog-widgets and set-up their attributes and events. The code-generator can also generate script-code that is necessary for an event-handling skeleton structure. This generated script is already runable without any changes - only the operational code that should be executed when the different events happen, need to be coded manually into this skeleton structure.

Numerous options make the code generator very flexible in its generation of code
Thereby, the program supports two basically different possibilities, how the script code is generated. On the one hand, the code generator can generate the dialog's creation code in core RapidBATCH script code, by generating the necessary NEWDIALOG- and LETDIALOG-instructions for creating and setting up the widgets and their attributes.
On the other hand, the dialog can also be generated from a dialog description file (file extension: .RBD), that is created by the Dialog Designer when the dialog is saved to a file. This generation of the dialog is performed by the RapidBATCH function GENDLG defined in the extending DIALOG.RB library. In this case, the code-generator generates the RapidBATCH code for importing and calling this function instead of the core creating functions.

Both methods have their advantages and disadvantages. By generating the dialog in core RapidBATCH script-code, extensions on this dialog need to be made manually later, but the advantage is, that you don't need to pass on the .RBD-file for the dialog when you want to distribute this script as a stand-alone .EXE-file.
When generating to a .RBD-file, the dialog is very quickly extendible and can be changed more easier. Also, less code is generated than in core-method, because NEWDIALOG- and LETDIALOG-instructions are ommitted. Using the RBD-file method is more efficient for greather applications, the core-method for smaller programs and tools.

Which method you prefer for your dialogs is on your own.

Next to these methods that are only necessary to the rhine generation and widget-setup of the dialog, the code-generator can also, as mentioned, generate a skeleton structure for all events that can happen with or on the different widgets. Here, when switchting this option on, you can select between three different event-handling models the code-generator can generate.
These models are demonstrated in the code examples below that where generated using the code generator for a simple dialog with a button and a small menu bar, using the core-method for the dialog creation.
  • IF...ELSEIF...ENDIF-Model
    Using this model, an IF- or ELSEIF-statement is generated for each event that can happen, within the event handling iteration. The code that should be executed can be inserted in each of the generated IF/ELSEIF-blocks. This is the most popular method that is used when implementing event handlers for individual dialog windows.

    rem --- creating code for widget "myDialog"
    newdialog 'myDialog', 'DIALOG', '220|100|384|133'
    letdialog 'myDialog', 'caption', 'Simple event demo'
    letdialog 'myDialog', 'menu', '&File:&About...|-|&Exit;'

    rem --- creating code for widget "myWidget1"
    newdialog 'myDialog:myWidget1', 'BUTTON', '1|1|373|76'
    letdialog 'myDialog:myWidget1', 'enabled', '0'
    letdialog 'myDialog:myWidget1', 'caption', 'Please click me!'

    rem --- make the dialog window visible
    letdialog 'myDialog', 'visible', [true]

    repeat
        rundialog [event] = '0'

        rem --- event handling for menu item "&About..."
        if [event] = 'click_myDialog:Menu_&About...'
            rem *** insert event code here ***

        rem --- event handling for menu item "&Exit"
        elseif [event] = 'click_myDialog:Menu_&Exit'
            rem *** insert event code here ***

        rem --- event handling for click on "myWidget1"
        elseif [event] = 'click_myDialog:myWidget1'
            rem *** insert event code here ***

        endif

    until [event] = 'close_myDialog'
    end
  • GOSUB...RESUME-Model
    Using this model, a label is generated to each event that can be called over a dynamic GOSUB-call.

    rem --- creating code for widget "myDialog"
    newdialog 'myDialog', 'DIALOG', '220|100|384|133'
    letdialog 'myDialog', 'caption', 'Simple event demo'
    letdialog 'myDialog', 'menu', '&File:&About...|-|&Exit;'

    rem --- creating code for widget "myWidget1"
    newdialog 'myDialog:myWidget1', 'BUTTON', '1|1|373|76'
    letdialog 'myDialog:myWidget1', 'enabled', '0'
    letdialog 'myDialog:myWidget1', 'caption', 'Please click me!'

    rem --- make the dialog window visible
    letdialog 'myDialog', 'visible', [true]

    repeat
        rundialog [event] = '0'

        if [event] ! 'close_myDialog'
            gosub [event]
        endif
    until [event] = 'close_myDialog'


    halt

    rem --- event handling for menu item "&About..."
    %click_myDialog:Menu_&About...
        rem *** insert event code here ***

    resume
    rem --- event handling for menu item "&Exit"
    %click_myDialog:Menu_&Exit
        rem *** insert event code here ***

    resume
    rem --- event handling for click on "myWidget1"
    %click_myDialog:myWidget1
        rem *** insert event code here ***

    resume

    end
  • Procedure-Model
    The procedure-model should only be used in greather projects. In this model, a procedure is created for each event. This procedure is then called from the event-loop using an IF/ELSEIF-construction, switching to the right procedure which is called there.

    rem --- creating code for widget "myDialog"
    newdialog 'myDialog', 'DIALOG', '220|100|384|133'
    letdialog 'myDialog', 'caption', 'Simple event demo'
    letdialog 'myDialog', 'menu', '&File:&About...|-|&Exit;'

    rem --- creating code for widget "myWidget1"
    newdialog 'myDialog:myWidget1', 'BUTTON', '1|1|373|76'
    letdialog 'myDialog:myWidget1', 'enabled', '0'
    letdialog 'myDialog:myWidget1', 'caption', 'Please click me!'

    rem --- event handling procedure for menu item "&About..."
    proc myDialog_Menu_About_click
        rem *** insert event code here ***
    endproc

    rem --- event handling procedure for menu item "&Exit"
    proc myDialog_Menu_Exit_click
        rem *** insert event code here ***
    endproc

    rem --- event handling procedure for click on "myWidget1"
    proc myDialog_myWidget1_click
        rem *** insert event code here ***
    endproc

    rem --- make the dialog window visible
    letdialog 'myDialog', 'visible', [true]

    repeat
        rundialog [event] = '0'

        if [event] = 'click_myDialog:Menu_&About...'
            myDialog_Menu_About_click
        elseif [event] = 'click_myDialog:Menu_&Exit'
            myDialog_Menu_Exit_click
        elseif [event] = 'click_myDialog:myWidget1'
            myDialog_myWidget1_click
        endif

    until [event] = 'close_myDialog'
    end
