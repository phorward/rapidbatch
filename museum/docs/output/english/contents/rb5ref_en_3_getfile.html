GETFILE
 
Syntax: GETFILE list-of-charcodes = filename, start-position, number-of-bytes
Description: Reads data from a file at binary-level.

Next to the filename, this function expects a starting position (byte-position where the binary read should beginn) and a length of bytes that should be read (number-of-bytes-parameter) from start-position.The returned value is a list of ASCII-character codes, where each code is one byte (= character) in the file. This list uses the RapidBATCH list separation character that is specified in the variable [std_sep], and by default a pipe-character (|).
Examples: rem example for writing and reading a file at byte-level
newfile 'test.txt'
rem write: "Hello World"
putfile 'test.txt', '1', '72|101|108|108|111|32|87|111|114|108|100'

rem read the word "World" and convert it to plain text
getfile [word] = 'test.txt', '7', '5'
echo 'List of ascii-charcodes: ' # [word]

rem converting routine
[i] = '0'
repeat
    [i] + '1'
    gettok [ch] = [word], '|', [i]
    if [ch] ! ''
        getchr [ch] = [ch]
        [plainword] # [ch]
    endif
until [ch] = ''

rem display the plain text
echo 'Plain text: ' # [plainword]
Remarks: Use the READFILE-function to read a text file's content line-by-line and in plain-text.
