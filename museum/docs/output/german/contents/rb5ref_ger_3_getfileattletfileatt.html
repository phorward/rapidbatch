GETFILEATT, LETFILEATT
 
Syntax: GETFILEATT Attributschalter = Dateiname, Attributnummer
LETFILEATT Dateiname, Attributnummer
Beschreibung: GETFILEATT gibt Auskunft, ob ein angefordertes Dateiattribut bei der ebenfalls spezifizierten Datei gesetzt ist oder nicht. Mögliche Werte für die Dateiattributnummer sind NORMAL (0), ARCHIVED (1), READONLY (2), COMPRESSED (3), DIRECTORY (4), HIDDEN (5) und SYSTEM (6) (zu Deutsch: Normal, Archiviert, Schreibgeschützt, Komprimiert, Verzeichnis, Versteckt und Systemdatei). Neben den numerischen Werten lassen sich optional auch die englischen Bezeichner als Attribut-Typ angeben, z.B. ARCHIVED oder READONLY.
GETFILEATT gibt 0 zurück, wenn das abgefragte Dateiattribut gesetzt ist, andernfalls -1.

LETFILEATT setzt Dateiattribute, wenn diese nicht gesetzt sind. Mögliche Werte für die Dateiattribute sind identisch zu denen von GETFILEATT, allerdings lassen sich die Attribute COMPRESSED (3) und DIRECTORY (4) nicht setzen.
Schlägt LETFILEATT fehl, so erhält [errorcode] den Wert -1, ansonsten 0.
Beispiele: [file] = '..\readme.txt'

rem Attribute auslesen
[i] = '1'
repeat
    getfileatt [ret] = [file],[i]
    [res] = [res] # [i] # ' = ' # [ret] # [new_line]
    [i] + '1'
until [i] > '6'

echo 'File Attributes of ' # [file] # [new_line] # [new_line] # [res]

rem Attribute setzen
letfileatt [file], 'normal'

end
