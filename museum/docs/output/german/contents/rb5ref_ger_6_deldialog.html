DELDIALOG
 
Syntax: DELDIALOG Widget-Label
Beschreibung: Entfernt ein Widget aus dem Speicher.

Wird als Widget-Label ein Dialogfenster angegeben, werden alle darauf befindlichen (= untergeordneten) Widgets automatisch mit entfernt.
Beispiele: rem Dialogfenster erzeugen
newdialog 'Fenster', 'dialog', '1|1|640|480'

rem Dialogtitel setzen und sichtbar machen
letdialog 'Fenster', 'caption', 'Ein Dialog'
letdialog 'Fenster', 'visible', [true]

rem Warten bis Benutzer den Schlieﬂen-Button drückt...
rundialog [ereignis] = '0'

rem Dialog aus dem Speicher entfernen
deldialog 'Fenster'

rem Versuch, das caption-Attribut zu lesen
getdialog [titel] = 'Fenster', 'caption'

if [errorcode] = [false]
    echo 'Huch, der Dialog existiert gar nicht mehr!'
else
    echo 'Diese Meldung wird niemals eintreten ;)'
endif
Bemerkungen: Schlägt DELDIALOG fehl, erhält [errorcode] den Wert -1, ansonsten 0.

Die DELDIALOG-Anweisung hat einzig und allein den Zweck, den belegten Speicher eines Dialoges wieder freizugeben, der in einer Library oder Prozedur verwendet wurde. Um den Dialog erneut zu verwenden, muss er explizit neu erstellt werden. DELDIALOG entfernt ein Widget bzw. einen kompletten Dialog explizit aus dem Speicher! Dies ermöglicht das dynamische Verwalten von Dialogen in Prozeduren. DELDIALOG sollte daher nur verwendet werden, wenn ein Dialog in einer Prozedur bei jedem Prozeduraufruf neu erstellt wird.
